## 📊 現状の問題点分析

### 1. 画像の問題（🔴 最優先）

#### 問題点
- **画像の取得方法が複雑**: `wikimedia-images.json`、`content.json`、Unsplashフォールバックの3層構造
- **画像の一貫性不足**: 一部の温泉地のみ画像があり、他はフォールバック画像
- **画像の品質**: Wikimedia Commonsの画像は古いものや解像度が低いものがある
- **画像の最適化不足**: Next.js Imageコンポーネントは使用しているが、画像URLの最適化が不十分

#### 具体的な問題
```typescript
// 現在の画像解決ロジック（複雑）
getOnsenImage() → wikimedia-images.json → フォールバック → Unsplash
```

**問題のある画像URL例**:
- `content.json`に直接記載されているURLが古い可能性
- Wikimedia Commonsの画像が適切でない場合がある
- Unsplashフォールバックが不適切な画像を返す可能性

### 2. ページコンテンツの不備（🔴 最優先）

#### 問題点
- **詳細情報の不足**: `content.json`の`docs`配列に詳細情報が不足
- **「準備中」状態**: 多くのページが「※ 詳細情報は準備中です。」のまま
- **温泉データの不足**: 泉質、効能、アクセス情報、営業時間などが不足
- **コンテンツの一貫性**: 一部のページのみ詳細情報があり、他は基本的な情報のみ

#### 具体的な問題
```json
// content.jsonの現在の状態
{
  "docs": [
    {
      "slug": "hakone",
      "title": "箱根温泉郷完全ガイド",
      "content": "## 箱根温泉の魅力\n\n箱根は年間約2,000万人が訪れる..."
      // ✅ 詳細情報あり
    },
    {
      "slug": "kusatsu",
      "content": "## 草津温泉について\n\n※ 詳細情報は準備中です。"
      // ❌ 詳細情報不足
    }
  ]
}
```

### 3. ベンチマークサイト「あえの風」との差

#### 「あえの風」の特徴
1. **高品質な画像**: すべてのページで一貫した高品質な画像
2. **充実したコンテンツ**: 詳細な情報、体験談、おすすめプラン
3. **視覚的な魅力**: 大型カード、パララックス効果、洗練されたアニメーション
4. **ユーザー体験**: 直感的なナビゲーション、検索機能、フィルタリング

#### 現在のサイトとの差
- ✅ デザイン要素は実装済み（Hero、カード、アニメーション）
- ❌ 画像の品質と一貫性が不足
- ❌ コンテンツの充実度が不足
- ❌ ユーザー体験の機能が不足（検索、フィルタリング）

```

```

#### 1.2 コンテンツの充実度
- **必須情報**: 泉質、効能、アクセス、営業時間
- **推奨情報**: 歴史、特徴、おすすめの過ごし方、周辺観光
- **オプション情報**: 体験談、レビュー、おすすめプラン

### 2. 画像戦略

#### 2.1 画像の種類
- **Hero画像**: 1920x1080px以上、16:9アスペクト比
- **ギャラリー画像**: 1200x800px以上、複数枚
- **サムネイル画像**: 400x300px、一覧表示用

#### 2.2 画像の選定基準
- **温泉が写っている**: 露天風呂、室内風呂、源泉など
- **高品質**: 解像度が高く、鮮明
- **適切なライセンス**: CC BY-SA 4.0推奨
- **一貫性**: すべての温泉地で同じスタイル

### 3. ユーザー体験戦略

#### 3.1 ナビゲーション
- **明確な階層**: トップ → エリア → 詳細ページ
- **検索機能**: 温泉地、泉質、効能での検索
- **フィルタリング**: エリア、日帰り可能、泉質でのフィルタ

#### 3.2 コンテンツの見せ方
- **大型カード**: 600px以上の高さで画像を大胆に使用
- **パララックス効果**: スクロール時の視覚的効果
- **アニメーション**: ホバー効果、スクロールアニメーション

### 4. SEO戦略

#### 4.1 メタデータ
- **タイトル**: 「[温泉地名] 温泉 | 関東温泉紀行」
- **ディスクリプション**: 150文字以内、キーワードを含む
- **OGP画像**: 1200x630px、各ページで異なる画像

#### 4.2 構造化データ
- **LocalBusiness**: 温泉地の基本情報
- **TouristAttraction**: 観光スポット情報
- **Review**: レビュー情報（オプション）

### 5. パフォーマンス戦略

#### 5.1 画像の最適化
- **Next.js Image**: 自動最適化
- **WebP/AVIF**: モダンフォーマットの使用
- **Lazy Loading**: 画面外の画像は遅延読み込み

#### 5.2 コードの最適化
- **コード分割**: ページごとのコード分割
- **Tree Shaking**: 未使用コードの削除
