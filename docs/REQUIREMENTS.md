# 要件定義書 (REQUIREMENTS.md)

## 1. プロジェクト概要

### 1.1 目的
*   **没入感のある温泉紹介サイトの構築**: ベンチマークサイト（あえの風）レベルの視覚的クオリティとユーザー体験を実現する。
*   **汎用的なサイト量産エンジンの開発**: テーマ（JSON設定）を切り替えるだけで、全く異なる業種・デザインのサイトを生成できるプラットフォームとする。
*   **JSON Firstアーキテクチャの確立**: コンテンツと構造を完全に分離し、運用効率と拡張性を最大化する。

### 1.2 スコープ
*   **フェーズ1**: 土台作り（リポジトリ整理、ルール定義）
*   **フェーズ2**: コアシステム再構築（型定義、画像システム刷新）
*   **フェーズ3**: 新デザイン実装（没入感のあるUI、大改造）
*   **フェーズ4**: コンテンツ統合・完成（リリースレベルへの仕上げ）

## 2. デザイン・UI要件

### 2.1 コンセプト
*   **没入感 (Immersive)**: 画面全体を使ったレイアウト、美しい写真、スムーズなアニメーションでユーザーを引き込む。
*   **脱・Webサイト感**: 既存の「ヘッダー＋コンテナ＋フッター」という構成を捨て、アプリや映像作品のようなUIを目指す。
*   **写真主役**: コンテンツの魅力を最大化するため、高品質な実写画像を大胆に使用する。

### 2.2 具体的なUI仕様
*   **ナビゲーション**:
    *   固定ヘッダーの廃止。
    *   スクロール連動型の透過ナビゲーション、または全画面メニュー（ハンバーガー）を採用。
    *   予約/CVボタンはフローティングまたは常時表示でアクセシビリティを確保。
*   **レイアウト**:
    *   `max-w-7xl` 等のコンテナ制限を原則撤廃。
    *   フルブリード（全幅）レイアウトを基本とする。
    *   非対称グリッドや大胆な余白を活用し、雑誌のようなレイアウトを実現。
*   **インタラクション**:
    *   シームレスなページ遷移。
    *   スクロールに応じたパララックス効果や要素のフェードイン。
    *   リッチなホバーエフェクト。

## 3. システム・技術要件

### 3.1 アーキテクチャ
*   **フレームワーク**: Next.js (App Router)
*   **スタイリング**: Tailwind CSS
*   **データ管理**: JSON First
    *   `content.json`: 構造・コンテンツデータ
    *   `texts.json`: UIラベル・固定テキスト
    *   `theme.json`: デザイン設定（配色、フォント、余白など）
*   **デプロイ**: GitHub Pages (静的エクスポート `output: 'export'`)

### 3.2 画像システム
*   **抽象化**: 画像パスをハードコードせず、「画像キー」で管理する。
*   **拡張性**: 将来的な外部API連携やAI自動収集に対応できる疎結合な設計とする。
*   **最適化**: `next/image` を活用し、レスポンシブ対応とフォーマット最適化を自動で行う。

### 3.3 テーマ機能
*   **マルチテーマ対応**: `themes/` ディレクトリ配下に複数のテーマを保持可能にする。
*   **抽象化されたコンポーネント**: 特定の業種（温泉）に依存しないコンポーネント設計（例: `OnsenCard` ではなく `FeatureCard`）。

## 4. 非機能要件

*   **パフォーマンス**: Lighthouseスコア 90以上（特にPerformanceとAccessibility）。
*   **品質**: `npm run build` および `npm run lint` が常にパスすること。
*   **保守性**: 厳格な型定義（TypeScript）とドキュメントによる運用。

*   **ログ**: 作業内容は必ず `WORK_LOG.md` に記録する。
