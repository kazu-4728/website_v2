### タスク1.2: 複数フリー画像APIを使用した画像検索の精度向上（次フェーズ）

**目的**: 残り15箇所の温泉地について、複数のフリー画像APIを使用して精度の高い温泉画像を取得

**対象API**:
1. **Wikimedia Commons API**（無料、登録不要）
   - 検索キーワード: `{温泉地名} Onsen rotenburo`, `{温泉地名} 温泉 露天風呂`
   - 優先度: 最高（著作権情報が明確）

2. **Pixabay API**（無料、登録必要）
   - 検索キーワード: `{温泉地名} onsen`, `{温泉地名} hot spring`
   - 優先度: 高（高品質な画像が多い）

3. **Pexels API**（無料、登録必要）
   - 検索キーワード: `{温泉地名} onsen`, `{温泉地名} hot spring`
   - 優先度: 中（多様な画像がある）

4. **Unsplash API**（無料、登録必要）
   - 検索キーワード: `{温泉地名} onsen`, `{温泉地名} hot spring`
   - 優先度: 中（レート制限が厳しい）

5. **Pexels Video API**（無料、登録必要）
   - 動画から静止画を抽出する可能性も検討

**実装内容**:

1. **マルチAPI検索スクリプトの改善**
   - ファイル: `scripts/fetch-onsen-images-multi-api-enhanced.js`
   - 複数のAPIを順番に試行
   - 各APIで複数の検索キーワードを試行
   - 画像の品質評価（解像度、アスペクト比、ファイルサイズ）

2. **画像品質評価ロジック**
   ```javascript
   function evaluateImageQuality(image) {
     // 1. 解像度チェック（最低1920x1080推奨）
     // 2. アスペクト比チェック（16:9推奨）
     // 3. ファイルサイズチェック（適切なサイズ）
     // 4. 画像タイプチェック（JPEG/PNG）
     // 5. 温泉関連キーワードの含有チェック
   }
   ```

3. **検索キーワードの最適化**
   - 各温泉地の特徴に合わせたキーワードを追加
   - 日本語と英語の両方を使用
   - 温泉関連のキーワードを優先（rotenburo、露天風呂、bath、浴場）

4. **画像の検証とフィルタリング**
   - 実際に温泉が写っている画像のみを選択
   - 画像のメタデータ（タイトル、説明、タグ）を確認
   - 不適切な画像を自動的に除外

**対象温泉地**:
- 鴨川、秩父、湯河原、強羅、中禅寺、勝浦、大洗、神川、奥多摩、仙石原、白浜、養老渓谷、袋田、瀬音の湯、塩原

**検証**:

- 著作権情報が正しく記録されていることを確認
