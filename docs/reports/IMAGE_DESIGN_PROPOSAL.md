# 画像管理設計の見直し提案

## 🔍 現状の問題

### 1. 画像の現状
- **全ての画像が同じUnsplash画像ID（photo-1583001931096）になっている**
- これは前回の修正で全て同じ画像に変更してしまったため
- 温泉らしい画像ではない可能性がある

### 2. 画像管理の仕組み
- `app/lib/images.ts`に画像管理システムがあるが、これは**GitHub Docsテーマ用**の画像
- 実際には`themes/onsen-kanto/content.json`内の画像URLが**直接使用**されている
- **テーマに合わせた自動取得の仕組みは存在しない**

### 3. 画像の取得元
- `content.json`内の`image`フィールドに直接URLが記述されている
- 各ページコンポーネントが`content.json`から画像URLを読み込む
- `app/lib/images.ts`は使用されていない（温泉テーマでは）

## 📋 設計の選択肢

### 選択肢1: content.jsonに適切な画像URLを設定（推奨）

**メリット**:
- シンプルで明確
- JSON-firstの設計に合致
- 各温泉地に適した画像を個別に設定可能

**デメリット**:
- 手動で画像URLを設定する必要がある
- 画像の管理が分散する

**実装方法**:
- `content.json`の各`image`フィールドに適切なUnsplash画像URLを設定
- 各温泉地に合った画像を選定

### 選択肢2: 画像管理システムを拡張

**メリット**:
- 画像を一元管理できる
- テーマごとに画像を切り替え可能

**デメリット**:
- 既存の設計を大きく変更する必要がある
- 複雑になる

**実装方法**:
- `app/lib/images.ts`を拡張して温泉テーマ用の画像を追加
- `content.json`から画像キーを参照する方式に変更

### 選択肢3: ハイブリッド方式

**メリット**:
- デフォルト画像を設定しつつ、個別に上書き可能
- 柔軟性が高い

**デメリット**:
- 実装が複雑になる

**実装方法**:
- `content.json`に画像キーを設定
- `app/lib/images.ts`でデフォルト画像を管理
- `content.json`にURLが設定されていればそれを優先

## 🎯 推奨案：選択肢1（content.jsonに適切な画像URLを設定）

### 理由
1. **JSON-firstの設計に合致**：既存の設計思想と一致
2. **シンプル**：追加のコード変更が不要
3. **明確**：画像の出所が明確
4. **柔軟性**：各温泉地に適した画像を個別に設定可能

### 実装手順

1. **各温泉地に適した画像URLを選定**
   - 箱根温泉：箱根の風景、温泉、富士山など
   - 草津温泉：草津の湯畑、温泉街など
   - 鬼怒川温泉：鬼怒川の渓谷、温泉など
   - その他の温泉地も同様に

2. **Unsplashから適切な画像を検索**
   - 検索キーワード例：
     - "hakone onsen japan"
     - "kusatsu hot spring japan"
     - "kinugawa onsen japan"
   - または、既存の温泉画像のIDを探す

3. **content.jsonを更新**
   - 各`image`フィールドに適切なURLを設定

### 画像URLの形式

```json
{
  "image": "https://images.unsplash.com/photo-[IMAGE_ID]?q=80&w=2070&auto=format&fit=crop"
}
```

## 🔧 次のステップ

1. **画像の選定**
   - 各温泉地に適した画像を選定
   - Unsplashで検索して適切な画像IDを特定

2. **content.jsonの更新**
   - 各`image`フィールドを更新

3. **テスト**
   - ビルドして画像が正しく表示されるか確認

## ❓ 確認事項

1. **画像の選定方法**：
   - Unsplashから手動で選定するか？
   - 特定のキーワードで自動検索するか？
   - 既存の画像ライブラリを使用するか？

2. **画像の統一性**：
   - 全ての画像を同じスタイル（例：風景写真）にするか？
   - 各温泉地の特色を反映した画像にするか？

3. **画像の品質**：
   - 高解像度の画像を使用するか？
   - 画像サイズの最適化は必要か？

## 📝 提案

まずは**選択肢1**で進めることを提案します。理由：
- 既存の設計を維持できる
- 実装が簡単
- 各温泉地に適した画像を設定できる

画像の選定については、Unsplashで各温泉地のキーワードで検索して、適切な画像IDを特定することをお勧めします。
