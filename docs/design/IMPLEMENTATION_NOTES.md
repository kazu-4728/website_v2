# 実装ノート - V5大改造

作成日: 2025年12月10日

## 📋 実装方針

### 基本方針
1. **既存サイトを完全に切り離す**: 既存のページ構成、コンポーネント名を引き継がない
2. **画像APIを活用**: 温泉画像に絞って取得
3. **拡張可能な構造**: ページ数も拡張可能に
4. **実装を素早く**: 状況に応じて変更しながら進める

### 画像APIの活用
- **Gemini API**: 環境変数から利用可能（chatやリポジトリに絶対出さない）
- **その他のAPI**: 順次実装（chatを切り替えないと環境変数から読み込めないため）
- **温泉画像に絞る**: 画像の使い方が重要

### ページ構成（拡張可能）
- 現在: ホーム + 温泉地詳細
- 将来: プラン、記事、お問い合わせを追加可能

## 🎯 実装の進め方

### フェーズ1: 設計の確定と準備（最優先）
- JSONスキーマの拡張
- 型定義の更新
- コンポーネント構成の確定

### フェーズ2: ホームページの大改造
- フルスクリーンHero（複数画像スライダー）
- エリア選択（大画像カード）
- おすすめ温泉（画像ギャラリー）
- 検索・絞り込み機能
- 温泉地一覧（大画像カード）

### フェーズ3: 温泉詳細ページの大改造
- フルスクリーンHero
- 画像ギャラリー
- 宿泊施設一覧
- 周辺観光スポット
- 予約導線

## 📝 実装メモ

### 画像の使い方
- フルスクリーンHero: 画面全体を使用
- 大画像カード: 画像を大胆に配置
- 画像ギャラリー: 複数画像のスライダー
- 背景画像: パララックス効果

### 拡張可能な構造
- JSON First: すべてのコンテンツをJSONで管理
- コンポーネントの汎用化: テーマに依存しない設計
- ページの追加: 新しいページタイプを簡単に追加可能
