目標: 画像メインの情報多彩な温泉紹介サイト

## 📋 設計方針

### 基本方針

1. **既存サイトを完全に忘れる**: 既存のページ構成、コンポーネント名、セクション構成を一切引き継がない
2. **ゼロから再設計**: 必要な機能だけを洗い出し、最適な構成を考える
3. **画像メイン**: 高品質な実写画像を大胆に使用
4. **情報多彩**: 各温泉地の詳細な情報を提供
5. **JSON First**: すべてのコンテンツとUIテキストをJSONで管理（維持）
6. **テンプレート化**: テーマを変更すれば他サイトに流用可能な構成

### ベンチマークサイトの参考点（デザイン例として）

1. **視覚的インパクト**
   - フルスクリーンHero（画面全体を使用）
   - 高品質な実写画像の大胆な使用
   - 複数画像のスライダー・ギャラリー
   - 画像にオーバーレイテキスト（グラデーション、タイポグラフィ）

2. **情報設計**
   - 明確な予約導線
   - 詳細な情報提供
   - アクセス情報の充実

3. **ユーザー体験**
   - スムーズなアニメーション
   - ホバーエフェクト
   - 視覚的リズム

**注意**: 「あえの風」はデザイン例として参考にするだけで、全て当てはめる必要はない

## 🎯 必要な機能の洗い出し（ゼロから考える）

### このサイトの目的
**関東の温泉地を紹介するサイト**

### ユーザーが求めていること
1. **温泉地を見つける**: どこにどんな温泉があるか
2. **温泉地を知る**: 各温泉地の詳細情報（泉質、効能、アクセス、宿泊施設）
3. **温泉地を選ぶ**: 自分の目的に合った温泉地を選ぶ
4. **予約する**: 宿泊施設を予約する

### 必要な機能
1. **温泉地の一覧表示**: すべての温泉地を見せる
2. **温泉地の詳細表示**: 各温泉地の詳細情報
3. **検索・絞り込み**: エリア、泉質、効能などで絞り込む
4. **予約導線**: 宿泊施設への予約リンク

## 🎨 新しいサイト構成（ゼロから設計）

### シンプルな構成案

```
/ (ホーム)
├── Hero（フルスクリーン、複数画像スライダー）

/[slug] (温泉地詳細)

├── 基本情報（泉質、効能、アクセス）
```

### より詳細な構成案

```

├── Hero（フルスクリーン、複数画像スライダー、大胆なオーバーレイ）

/[slug] (温泉地詳細)

├── 概要・魅力（大画像 + テキスト）
├── 温泉データ（アイコン付きカード）
├── 画像ギャラリー（複数画像のスライダー）
├── 宿泊施設一覧（大画像カード、予約ボタン）
├── 周辺観光スポット（大画像カード）
├── 地図（インタラクティブ）
├── 季節ごとの魅力（季節ごとの画像）
└── 予約ボタン（明確なCTA）
```

## 🖼️ 画像メインのデザイン

### 画像の使い方

1. **フルスクリーンHero**
   - 画面全体を使用した大胆なビジュアル
   - 複数画像のスライダー（自動切り替え、手動切り替え）
   - オーバーレイテキスト（グラデーション、タイポグラフィ）

2. **大画像カード**
   - 各セクションで画像を大胆に配置
   - ホバーエフェクト（ズーム、リフト、シャドウ）
   - 画像の上にテキストを配置（オーバーレイ）

3. **画像ギャラリー**
   - 複数画像のスライダー
   - サムネイル表示
   - ライトボックス表示

### 画像の配置原則

- **大胆に**: 画像を大きく、画面全体を活用
- **複数画像**: 各セクションで複数の画像を表示
- **視覚的階層**: 大きな画像、適切な余白、タイポグラフィの統一
- **品質重視**: 高品質な実写画像のみを使用

## 🎯 テンプレート化の考慮

### テーマ切替可能な設計

1. **コンテンツ構造の統一**
   - `content.json`のスキーマを統一
   - テーマごとに`content.json`を切り替え可能

2. **画像システムの統一**
   - 画像キー→URL解決の仕組みを維持
   - テーマごとに画像マッピングを定義

3. **UIテキストの統一**
   - `texts.json`のスキーマを統一
   - テーマごとに`texts.json`を切り替え可能

4. **コンポーネントの汎用化**
   - テーマに依存しないコンポーネント設計
   - JSONからデータを読み込んで表示
