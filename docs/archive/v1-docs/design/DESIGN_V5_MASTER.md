# 設計構想 V5 - ベンチマークサイトを目指す大改造

作成日: 2025年12月10日  
ベンチマーク: 和倉温泉 あえの風 公式サイト（https://www.aenokaze.jp/）  
目標: 画像メインの情報多彩な温泉紹介サイト

## 📋 設計方針

### 基本方針

1. **ベンチマークサイトを目指す**: 「あえの風」レベルの世界観・クオリティを実現
2. **画像メイン**: 高品質な実写画像を大胆に使用
3. **情報多彩**: 各温泉地の詳細な情報を提供
4. **斬新なデザイン**: 既存サイトを忘れ、ゼロから再設計
5. **JSON First**: すべてのコンテンツとUIテキストをJSONで管理（維持）
6. **テンプレート化**: テーマを変更すれば他サイトに流用可能な構成

### ベンチマークサイトの核心要素

1. **視覚的インパクト**
   - フルスクリーンHero（画面全体を使用）
   - 高品質な実写画像（温泉、客室、食事）
   - 複数画像のスライダー・ギャラリー
   - 画像にオーバーレイテキスト（グラデーション、タイポグラフィ）
   - 視覚的階層（大きな画像、適切な余白）

2. **情報設計**
   - 明確な予約導線（ヘッダーに常時表示、各ページに予約ボタン）
   - 詳細なプラン情報（料金、日程、含まれるもの、キャンセルポリシー）
   - 施設の詳細（客室、温泉、食事、設備）
   - アクセス情報（地図、交通手段、駐車場、周辺情報）

3. **ユーザー体験**
   - スムーズなスクロールアニメーション（パララックス効果）
   - ホバーエフェクト（画像ズーム、カードリフト、シャドウ）
   - マイクロインタラクション（ボタン、リンクの細かなアニメーション）
   - 視覚的リズム（セクション間の適切な余白、視覚的な流れ）

## 🖼️ 画像メインのデザイン

### 画像の使い方

1. **フルスクリーンHero**
   - 画面全体を使用した大胆なビジュアル
   - 複数画像のスライダー（自動切り替え、手動切り替え）
   - オーバーレイテキスト（グラデーション、タイポグラフィ）

2. **大画像カード**
   - 各セクションで画像を大胆に配置
   - ホバーエフェクト（ズーム、リフト、シャドウ）
   - 画像の上にテキストを配置（オーバーレイ）

3. **画像ギャラリー**
   - 複数画像のスライダー
   - サムネイル表示
   - ライトボックス表示

4. **背景画像**
   - セクションの背景に画像を使用
   - パララックス効果
   - オーバーレイでテキストの可読性を確保

### 画像の配置原則

- **大胆に**: 画像を大きく、画面全体を活用
- **複数画像**: 各セクションで複数の画像を表示
- **視覚的階層**: 大きな画像、適切な余白、タイポグラフィの統一
- **品質重視**: 高品質な実写画像のみを使用

### コンテンツ構造

```json
{
  "site": {
    "name": "関東温泉紀行",
    "tagline": "Kanto Onsen Journey"
  },
  "pages": {
    "home": {
      "hero": {
        "type": "cinematic-fullscreen",
        "slides": [...],
        "overlay": {
          "gradient": "from-dark-950/80 via-dark-950/60 to-transparent",
          "textStyle": "bold-large"
        }
      },
      "sections": [
        {
          "type": "image-gallery",
          "title": "エリアから探す",
          "items": [...],
          "layout": "large-cards",
          "hoverEffect": "zoom-lift"
        }
      ]
    },
    "docs": {
      "items": [
        {
          "slug": "hakone",
```

## 🎯 テンプレート化の考慮

### テーマ切替可能な設計

1. **コンテンツ構造の統一**
   - `content.json`のスキーマを統一
   - テーマごとに`content.json`を切り替え可能

2. **画像システムの統一**
   - 画像キー→URL解決の仕組みを維持
   - テーマごとに画像マッピングを定義

3. **UIテキストの統一**
   - `texts.json`のスキーマを統一
   - テーマごとに`texts.json`を切り替え可能

4. **コンポーネントの汎用化**
   - テーマに依存しないコンポーネント設計
   - JSONからデータを読み込んで表示
