# 温泉紹介サイト設計仕様（草津三湯ベンチマーク反映）

更新日: 2026-03-27
対象プロジェクト: website_v2

## 1. 目的
本仕様は、温泉紹介サイトを「雰囲気訴求だけ」ではなく「来訪意思決定まで支援するサイト」に進化させるための実装基準を定義する。

## 2. UX方針
- ファーストビューは温泉写真を主役にする（大判、余白、低ノイズな情報量）。
- ページ中段で体験価値を伝え、後半で実務情報（利用案内・アクセス・FAQ）を提供する。
- 情報は「情緒 -> 判断 -> 行動」の順で配置する。

## 3. 情報アーキテクチャ
- トップページ
- 温泉一覧ページ (`/docs`)
- 温泉詳細ページ (`/docs/[slug]`)
- 補助ページ（将来拡張）
  - 利用案内（共通）
  - 交通案内（共通）
  - FAQ（共通）

## 4. 画面構成
### 4.1 トップページ
- 大判ヒーロー
- エリア別カード（温泉詳細への導線）
- 特集・体験訴求セクション

### 4.2 温泉一覧
- 温泉カード一覧（画像 + 短文 + CTA）
- カード導線は `/docs/{slug}` に統一

### 4.3 温泉詳細
- ヒーロー（画像 + タイトル + 要約）
- 本文（Markdown）
- 来訪ガイド（利用案内・アクセス・FAQ）
- 温泉データ（泉質、効能、日帰り可否、アクセス）
- 季節の魅力
- 前後ページ導線

## 5. コンポーネント設計
- `DocPageHero`
  - 役割: 感性的な第一印象を形成
  - 入力: `title`, `subtitle`, `description`, `heroImage`
- `GuideSection`
  - 役割: 来訪前の不安解消
  - 入力: `GuideCard[]`
- `DetailSection`
  - 役割: 構造化された事実情報表示
  - 入力: `title`, `children`
- `SeasonCards`
  - 役割: 季節別の再訪動機を提示

## 6. データ要件（最低限）
以下を温泉単位で保持する。

```json
{
  "slug": "hakone",
  "name": "箱根温泉",
  "heroImage": "https://...",
  "summary": "...",
  "guide": {
    "usage": ["営業時間", "料金", "設備"],
    "access": ["最寄り駅", "車", "駐車場"],
    "faq": ["タオル", "アメニティ", "入浴ルール"]
  },
  "onsen": {
    "springTypes": ["..."],
    "effects": ["..."],
    "dayTripAvailable": true
  }
}
```

## 7. 文言ポリシー
- 断定情報は公式情報ソースがあるもののみ掲載する。
- 不明値は推測しない（例: "施設により異なる" と明示）。
- FAQは問い合わせ削減を目的に、運用で頻出する質問を優先する。

## 8. 画像ポリシー
- 各温泉で最低1枚は「温泉・浴槽が明確に写る写真」を採用する。
- 景観写真だけで構成しない（実利用イメージを優先）。
- 画像クレジットまたは出典管理を行う。

## 9. 導線ポリシー
- 一覧 -> 詳細の導線は一意にする（`/docs/{slug}`）。
- 詳細ページ内に「利用案内」「アクセス」「FAQ」へのジャンプ導線を常設する。
- 前後ページ導線で回遊率を確保する。

## 10. 今回の実装反映（2026-03-27）
- 温泉一覧カード遷移を `/docs/{slug}` に統一。
- 詳細ページに来訪ガイドセクション（利用案内・アクセス・FAQ）を追加。
- 詳細ページの前後ナビも `/docs/{slug}` に統一。

## 11. 次フェーズ
1. 温泉ごとの実測データを `guide` に埋める。
2. 共通FAQページを実装し、詳細から双方向リンクする。
3. 多言語対応（英語・繁体・簡体）用の翻訳キーを定義する。
